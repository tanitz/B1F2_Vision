Input Image

รับภาพจากกล้อง / โหลดไฟล์ BMP

ระบบรับภาพจากกล้องอุตสาหกรรมหรือโหลดจากไฟล์ผ่าน OpenCV ภาพถูกอ่านเป็น NumPy array รูปแบบ BGR โดยไม่มีการแปลงหรือบีบอัดใดๆ

  • รูปแบบไฟล์: BMP, JPEG, PNG — BMP แนะนำเพราะไม่สูญเสียข้อมูล (lossless)
  • ความละเอียด: 2592 × 2048 px (5.0 MP) — ความละเอียดสูงช่วยให้ตรวจจับรายละเอียดชิ้นส่วนขนาดเล็กได้แม่นยำ
  • สี: BGR 3 channels (Blue-Green-Red) ตามมาตรฐาน OpenCV
  • การเข้าถึง: ภาพถูกเก็บใน RAM เป็น ndarray พร้อมนำไปประมวลผลต่อทันที

Downscale for Processing

ย่อภาพเพื่อเพิ่มความเร็ว Template Matching

ภาพต้นฉบับมีขนาดใหญ่เกินไปสำหรับ Template Matching โดยตรง ระบบย่อภาพก่อน matching แล้วแปลงพิกัดผลลัพธ์กลับไปยัง full-resolution สำหรับการวาด overlay

  • Scale factor: min(800 / max_side, 1.0) = 0.31x → ย่อจาก 2592×2048 เหลือ 800×632 px
  • Interpolation: INTER_AREA — เหมาะสมที่สุดสำหรับการย่อภาพ ลด aliasing
  • ผลประหยัด: พื้นที่ภาพลดลง 90% → matching เร็วขึ้น ~10x
  • Inverse scale: inv = 1 / 0.31 = 3.24x ใช้แปลง bounding box กลับเป็น full-resolution ก่อนวาดบนภาพจริง

Search ROI Definition

กำหนดพื้นที่ค้นหาของแต่ละ Inspection Point จาก Model JSON

แต่ละ Inspection Point มีขอบเขตการค้นหา (Search ROI) ที่กำหนดล่วงหน้าในไฟล์ JSON เพื่อจำกัดพื้นที่ที่ระบบจะค้นหา Template แทนการสแกนทั้งภาพ

  • โมเดล CV5VS_MODEL: มี 8 inspection points ได้แก่ High Pressure Sensor, Bolt & Washer, Hot Gas Valve, Common-Start-Run, Filter Drier, Check Valve, Expansion Valve
  • รูปแบบ ROI: [x, y, width, height] กำหนดเป็น pixel coordinates บนภาพ full-resolution
  • ลด False Positive: ป้องกันระบบจับชิ้นส่วนที่คล้ายกันในบริเวณที่ไม่เกี่ยวข้อง
  • เพิ่มความเร็ว: NCC คำนวณเฉพาะภายใน ROI แทนทั้งภาพ — ลดเวลาประมวลผลอย่างมีนัย
  • กำหนดครั้งเดียว: ROI ตั้งค่าตอนสร้างโมเดล (Settings) และใช้ซ้ำในทุก inspection

Image Pyramid Construction

สร้าง Gaussian Pyramid หลายระดับ — Coarse-to-Fine Strategy

Template Pyramid (ชุด Template ที่ใช้ค้นหา)

Image Pyramid คือการสร้างภาพชุดเดียวกันในหลายระดับความละเอียด เพื่อให้ระบบค้นหาตำแหน่งคร่าวๆ จากภาพเล็กก่อน แล้ว refine ลงมาที่ภาพใหญ่

  • วิธีสร้าง: cv2.pyrDown() — Gaussian blur แล้วลดขนาดครึ่งหนึ่งในแต่ละระดับ (ทั้ง Scene และ Template)
  • จำนวนระดับ: คำนวณอัตโนมัติจาก min_reduce_area = 256 — ยิ่ง template ใหญ่ยิ่งมีระดับมาก
  • Stage 1 (Coarse): ค้นหา candidate positions บนภาพระดับบนสุด (เล็กสุด) → เร็วมาก ได้ตำแหน่งโดยประมาณ
  • Stage 2 (Refine): นำ candidates ลงมา refine ทีละระดับจนถึงระดับ 0 (full-res) → ได้ตำแหน่งแม่นยำระดับ pixel
  • Score threshold ผ่อนคลาย: ระดับบน threshold = 0.50 × 0.9n เพื่อไม่ตัด candidates ที่ยังไม่แม่นจากภาพย่อ

NCC Template Matching

คำนวณ Normalized Cross-Correlation (TM_CCOEFF_NORMED) บน Search ROI

NCC วัดความคล้ายคลึงระหว่าง Template กับทุกตำแหน่งใน ROI ผลลัพธ์คือ Score Map ที่แต่ละ pixel แสดงค่าความตรงกับ Template ณ ตำแหน่งนั้น

  • Algorithm: cv2.TM_CCOEFF_NORMED — Normalized Cross-Correlation, ผล = –1 ถึง 1 (1.0 = identical)
  • Score Map ขนาด: (HROI – Htmpl + 1) × (WROI – Wtmpl + 1) แต่ละ pixel = score ณ ตำแหน่ง top-left ของ template
  • Heatmap (JET colormap): ■ แดง = score สูง (match ดี) / ■ น้ำเงิน = score ต่ำ (ไม่ตรง)
  • Peak detection: cv2.minMaxLoc() หา pixel ที่มี score สูงสุด → ตำแหน่งที่ตรงกับ Template มากที่สุด
  • Threshold: score ≥ 0.50 ถือว่า match — ค่าต่ำกว่านี้ถูกตัดทิ้ง
  • Validation: ผ่านการตรวจ Laplacian variance (กรองพื้นที่ไม่มี texture) และ SSIM ≥ 0.35 (กรอง false match จาก normalization artifact)

OK / NG Template Comparison

รัน FPM กับ Template ทั้งสองชุดแล้วเปรียบ score เพื่อตัดสิน

ระบบมี Template สองชุดต่อ Inspection Point — ชุด OK (ชิ้นส่วนปกติ) และชุด NG (ชิ้นส่วนผิดปกติ) การเปรียบ score ทั้งสองชุดทำให้ตัดสินได้แม่นยำกว่าใช้ threshold เพียงอย่างเดียว

  • OK Templates: ภาพตัวอย่างชิ้นส่วนที่ติดตั้งถูกต้อง หลายรูปเพื่อครอบคลุมความแตกต่างด้านแสง มุม และตำแหน่ง
  • NG Templates: ภาพตัวอย่างชิ้นส่วนที่ผิดปกติ เช่น หาย, ผิดรุ่น, ติดตั้งผิด, สายหลุด
  • Multi-template matching: แต่ละชุดอาจมีหลายรูป ระบบเลือก score สูงสุดจากทุกรูปในชุดนั้นมาเปรียบกัน
  • ตัวอย่างโมเดลนี้: Filter Drier มี 4 OK templates และ 3 NG templates
  • ข้อดี: แม้ชิ้นส่วนอื่นจะบังอยู่หรือแสงเปลี่ยน NG template ยังสามารถจับสภาพ defect ได้โดยตรง

Decision Logic

ตัดสิน PASS / FAIL / NO_MATCH จากการเปรียบ score_ok กับ score_ng

กฎการตัดสินผลของแต่ละ Inspection Point ใช้การเปรียบ score สองชุด เพื่อให้ระบบแยกแยะ OK กับ NG ได้แม้ score ของทั้งสองชุดใกล้เคียงกัน

  • PASS: score_ok ≥ 0.50 และ score_ok ≥ score_ng → ชิ้นส่วนอยู่ครบและตรงกับสภาพปกติมากกว่าสภาพผิดปกติ
  • FAIL: score_ng > score_ok → สภาพปัจจุบันตรงกับ NG template มากกว่า → แสดงว่าผิดปกติ
  • NO_MATCH: ทั้ง score_ok และ score_ng < 0.50 → ไม่พบชิ้นส่วนในพื้นที่ที่กำหนด → นับเป็น FAIL
  • Overall Result: ทุก Inspection Point PASS → ภาพนี้ PASS / มี FAIL อย่างน้อย 1 จุด → ภาพนี้ FAIL
  • ปรับ threshold: สามารถเปลี่ยนค่า 0.50 ได้ในโค้ด (score_threshold=0.50) เพื่อสมดุลระหว่าง sensitivity และ specificity

Result Visualization

วาด Bounding Box, Score และ Highlight บน Full-Resolution Image

ผลลัพธ์ทุก Inspection Point ถูกวาดทับบนภาพ full-resolution เพื่อให้ผู้ใช้เห็นว่าระบบตรวจพบชิ้นส่วนที่ไหน และมั่นใจมากน้อยเพียงใด

  • Rotated Bounding Box: กล่องสี่เหลี่ยมวาดตามมุมที่ template ตรงกัน รองรับการหมุน (tolerance_angle) แต่ปัจจุบัน = 0° (ไม่หมุน)
  • สี: ■ เขียว = PASS, ■ แดง = FAIL, ■ เทา = NO_MATCH
  • Score label: แสดง s=0.97 บนกล่องทุกจุด — ยิ่งใกล้ 1.0 ยิ่งมั่นใจ
  • Semi-transparent fill: fill_alpha=0.15 ไฮไลต์พื้นที่ที่ match เพื่อให้เห็นชัดขึ้น
  • บันทึก Result Image: ภาพ annotated ถูกบีบอัด JPEG (quality=85) และบันทึกใน results/images/ พร้อม timestamp ID

Save Report

บันทึกผลลัพธ์ลง CSV รายวัน (YYYY-MM-DD.csv) และ Result Image

ทุก inspection run จะถูกบันทึกอัตโนมัติในรูปแบบ CSV รายวัน สามารถเรียกดูผ่านหน้า Report ได้ทันที โดยระบบอ่านไฟล์ของวันนั้นๆ

  • ไฟล์ CSV: results/YYYY-MM-DD.csv — แยกรายวัน 1 ไฟล์ต่อวัน ง่ายต่อการ backup และ archive
  • 1 แถว = 1 Inspection Point ภาพ 1 ใบมีหลายแถว (เท่ากับจำนวน inspection points) และมี result_image_id เดียวกัน
  • คอลัมน์: timestamp · model · inspection_name · result · score · overall_result · result_image_id
  • Simulated Timestamp: เริ่ม 08:00:00 + 90 วินาที ต่อ inspection run เพื่อให้ข้อมูล timeline สมจริง
  • Result Image: บันทึกใน results/images/{timestamp}.jpg เชื่อมโยงกับแถว CSV ผ่าน result_image_id
  • หน้า Report: อ่าน CSV วันปัจจุบันโดยอัตโนมัติ แสดง PASS/FAIL พร้อมรูปผลลัพธ์
